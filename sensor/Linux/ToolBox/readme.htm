USB Driver Installation – Windows

The USB drivers are downloaded as part of the NDI ToolBox installation. The default download location is:

  • Windows XP and Vista 32 bit – C:\Program Files\Northern Digital Inc\ToolBox\USB Driver
  • Windows Vista 64 bit and Windows 7/8 – C:\Program Files (x86)\Northern Digital Inc\ToolBox\USB Driver

The drivers are also located on the installation CD, in the USBDrivers folder.

Note: Connect the USB Adapter before installing the drivers.

Install the drivers for Windows 7/8 as follows:

  1. When you first connect the NDI SCU or NDI Host USB Converter to the host computer, an error message may appear over the task bar. Ignore the message and complete the following steps to install the USB drivers
  2. (Windows 8 only) From the Start Screen, right-click and select All apps.
  3. From the Windows menu, select Control Panel.
  4. In the top right hand corner of the Control Panel window, select View by: Small Icons.
  5. From the Control panel, select Device Manager.
  6. Under Other Devices, right click NDI Host USB Converter.
  7. Select Update Driver Software... then select Browse my computer for driver software.
  8. Select Program Files (x86)\Northern Digital Inc\ToolBox\USB Driver. Select Next.
  9. In the Windows security dialog, select Install this driver software anyway option. The first set of drivers will install.
  10. After the first set of drivers are installed, browse to Other Devices and right click USB Serial Port.
  11. Select Update Driver Software... then select Browse my computer for driver software.
  12. Select Program Files (x86)\Northern Digital Inc\ToolBox\USB Driver. Select Next.
  13. In the Windows security dialog, select Install this driver software anyway option. The second set of drivers will install.

Driver installation is complete and the system will now appear to be connected through a virtual COM port.

Install the drivers for Windows XP and Vista as follows:

  1. When you first connect the NDI SCU or NDI Host USB Converter to the host computer, the Found New Hardware Wizard will begin automatically.
  2. When prompted, select the option that allows you to specify the drivers' location. ("Install from a specific list or location" in Windows XP and Windows Vista.)
  3. Browse to the folder containing the drivers, either on the host computer, or the installation CD. Windows will automatically select the correct drivers from the folder.
  4. Click Next and Finish as required to complete the installation.
    Note: After you have selected the folder containing the USB drivers, a warning may appear, indicating that the drivers have not passed. Select "Continue Anyway."
  5. Once the first set of drivers is installed, the Found New Hardware Wizard will begin a second time. Follow steps 2 to 4 above to install the second set of drivers.

Driver installation is complete and the system will now appear to be connected through a virtual COM port.

USB Driver Installation – Linux

The NDI USB drivers are included and built in on Linux kernel versions 2.6.32 and later available at www.kernel.org. Patching of these kernels is not required. NDI
has included patches for specific previous versions on the product CD. The USB patch files are downloaded as part of the NDI ToolBox installation. The default download
location is ~/NDIToolbox. The files are also located on the product CD, in the USBDrivers folder.

NDI systems appear as device files "/dev/ttyUSBx" (where x is enumerated). The driver emulates a standard serial port such that the USB device may be communicated with as a standard
RS-232 or RS-422 device.

Note: In order to access the system from a non-root user account, the account must be added to the "lock" and "dialout" groups. On some older systems it may also be necessary
to add it to the "tty" and "uucp" groups. This can be done with the command "sudo usermod -G lock,dialout <account name>"

If you wish to install and run ToolBox on a 64 bit system, you will have to install the 32 bit libraries first. On Ubunto and Debian systems this can be done with the command
sudo apt-get install ia32-libs

To Install the USB Patches:

The instructions below are based on the Linux 2.6.x kernel, where 2.6.x is the kernel version number. The following changes are captured in the ftdi.patch file. The patch
file is located in the <user_account>/usb-patch/ directory after NDI ToolBox has been installed.

Note: If you are using a different kernel level, the patch file may not work. Contact NDI for assistance.  Only apply the patch applicable to your kernel version. Do not apply
the patch more than once.

The patch modifies the files ftdi_sio.c and ftdi_sio.h in the directory /usr/src/linux-2.6.x/drivers/usb/serial. The modifications are:

1. ftdi_sio.c - added Polaris Spectra/Vicra SCU/Host USB Converter product ID to supported devices tables, added aliasing of 19.2 K baud for the Polaris Spectra or Polaris
Vicra System to use 1.2 MB, and optimized transmit latency setting.

2. ftdi_sio.h - added the define for the USB product ID for the Polaris Spectra/Vicra SCU/Host USB Converter.

3. Apply the patch to an unmodified kernel as follows:

1. Log on as root user.

2. cd /usr/src/linux-2.6.x.

3. create backups of the files ./drivers/usb/serial/ftdi_sio.c and ./drivers/usb/serial/ftdi_sio.h.

4. patch -p1 < ftdi-usb-patch-kernel-2.6.x.patch

5. If the driver is a kernel-loadable module, apply the patch as follows:

a)   make modules

b)   make modules_install

c)   Restart the computer.

If the driver is not a kernel-loadable module, rebuild the kernel following the instructions specific to the kernel. If you are unfamiliar with kernel rebuilding, refer to the
instructions usually located in the source directory or at www.kernel.org.

6. The SCU or Host USB Converter when connected will appear as “/dev/ttyUSBx”, where x is the port number.

Note: SCUs and Host USB Converters are interchangeable on the same USB port, without having to re-install drivers or losing the device file enumeration previously established,
as long as all serial converters (NDI or other FTDI-based converters) are connected and enumerated in the same order.

USB Driver Installation – Mac OS X

The USB driver required to connect to the Polaris Spectra and Polaris Vicra systems is copied to the "USB Driver" sub-folder of the ToolBox installation folder as part of the installation of ToolBox. The driver installer can also be found in the "USB Driver" sub-folder of the "MacOSX" folder on the installation CD.

For detailed instructions on USB driver installation please refer to the user guide that was delivered with the system. The instructions and the supplied driver have been tested with Mac OS X 10.5.8 on Intel-based systems.

Known Issues

  1. On Windows 7 if the CPU goes to sleep the USB driver may not wake up properly and communication will not resume unless the port is closed and re-opened.
  2. Linux kernel versions 2.6.33 and 2.6.34 contain a defect that renders the FTDI usb serial driver inoperative. These kernel versions cannot be used with the Spectra, Vicra and Aurora products. The defect is fixed in kernel 2.6.35 and later versions.
  3. On Mac OS X 10.5 there is an issue related to the Java 6 JVM and the USB driver that affects communications while performing an firmware update on hybrid Spectra and Aurora V2 systems. Communication errors occur which may cause the update to fail. When using these systems on OS X 10.5 you should install from file install-10_5.dmg. This will install an older version of ToolBox which runs on the Java 5 JVM. Please note: This issue does not affect passive-only Spectra/Vicra or Aurora V1 systems.
